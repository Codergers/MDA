# MDA

</div>

MDA全称是Maa Doro Assistant ，是[DoroHelper](https://github.com/1204244136/DoroHelper) 基于[MaaFramework](https://github.com/MaaXYZ/MaaFramework)重写而来的版本

> **MaaFramework** 是基于图像识别技术、运用 [MAA](https://github.com/MaaAssistantArknights/MaaAssistantArknights) 开发经验去芜存菁、完全重写的新一代自动化黑盒测试框架。
> 低代码的同时仍拥有高扩展性，旨在打造一款丰富、领先、且实用的开源库，助力开发者轻松编写出更好的黑盒测试程序，并推广普及。

# 使用与反馈须知

## 一、启动与操作说明

1. **启动后请先熟悉界面**
   不要急于执行任务，先浏览各项功能与设置，了解当前可用操作。

2. **未显示的功能即为未实现**
   请勿尝试使用界面中不存在的功能。

## 二、快捷键设置（重要）

- 进入：**右上角「设置」→「快捷键」**
- 操作：**开启全局快捷键**
- 推荐快捷键：`F11` 用于快速结束任务，防止程序卡死。

## 三、反馈前准备（必须执行）

### 1. 开启调试图像保存

- 路径：`设置` → `调试`
- 操作：勾选 **「保存调试图像」**
- ⚠️ 注意：**每次运行程序都需要重新开启此选项**

### 2. 调试模式使用建议（重要）

> 调试模式会为**每一个动作**保存一张图片，长时间运行会导致图片数量爆炸，占用大量空间。

**正确使用方式：**

- 开启调试模式后，**遇到一个问题就打包发送一次日志**
- 发送完成后 **重启程序**，再跑剩下的任务
- 推荐做法：**单独开启调试模式，只跑有问题的任务**

**重启程序后的关键步骤：**

- **必须删除原有的 `vision` 文件夹**
- 保证**每一个问题对应一个全新的 `vision` 文件夹**

**这样做的好处：**

- 方便开发人员定位修复问题
- 避免生成海量图片，控制日志文件大小
- 每个问题的调试图像独立清晰，不会混淆

## 四、导出与发送反馈日志（完整步骤）

1. 点击右下角 **「运行日志」** 右侧的 **「导出日志」** 图标
2. 系统会自动跳转到 `debug` 文件夹
3. 找到生成的 **压缩文件**，将其 **解压**（注意：不要打开解压后的文件）
4. 在相同目录下找到 **`vision` 文件**，将其 **拖入解压后的文件夹**
5. 确认该文件夹内包含以下内容：
   - `vision` 文件夹
   - `config` 文件夹
   - `XXXX-XX-XX-X.log`
   - `go-service.log`
   - `maafw.log`
6. 将 **整个文件夹重新压缩为 ZIP 格式**
7. 发送压缩包给开发人员

> ✅ 虽然操作稍显繁琐，但能极大提高问题定位效率，感谢配合！
